The Rules

  1. each showcase is a div
  2. they have two classes: "showcase" and "div#"
  3. the name of each will be an h2 element
  4. the css for .showcase is:
				
					.showcase {
						/* so that other elements can be positioned relative to it */
						position: relative;
						/* so that there is no spillover */
						overflow: hidden;
						/* giving space between them */
						margin-bottom: 2rem;
						/* making it look a little nicer */
						padding: 1rem;
						/* so I can see where they are */
						outline: 2px solid blue;
					}

My Thoughts on C

December 26, 2020

Scripting is the meat and potatoes of DevOps, and it's where I feel at ${HOME}. The toolsets available to scripters like myself have an altogether different set of requirements than other programming languages. A scripting language should be portable, interpreted, and infinitely extensible. Runtime performance and memory management are but afterthoughts in scripts which can take hours to complete. Bash fits the role perfectly, and it's my favorite language. However, as a firm believer in the Unix philosophy, it was time to put my money where my mouth is. It's why I chose to write my chess engine in C.
